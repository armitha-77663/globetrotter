Project Overview
GlobeTrotter is a travel planning web application designed to simplify and structure the process of organizing personalized trips. The platform focuses on providing users with an intuitive interface to create accounts, explore the application, and initiate travel plans efficiently.
This repository currently includes the foundational frontend pages required to onboard users and begin trip planning. The project is structured to allow future expansion into itinerary management, budgeting, activity discovery, and trip sharing.

Current Implementation Status
The following core pages have been implemented in this version of the application:
->Login Page
->Register Page
->Landing Page
->Plan My Trip Page
These pages establish the primary user flow from authentication to trip creation.

Implemented Features
1. Login Page
->Purpose: Authenticate existing users and provide secure access to the platform.
->Functionality:
Email and password input fields
Client-side validation
Login action handling
Navigation to registration and password recovery options

2. Register Page
->Purpose: Allow new users to create an account.
->Functionality:
User registration form
Input validation for required fields
Account creation workflow
Redirect to login after successful registration

3. Landing Page
->Purpose: Serve as the entry point and introduction to the platform.
->Functionality:
->Application overview and value proposition
->Call-to-action buttons (Login / Register / Plan Trip)
->Clean and responsive layout

4. Plan My Trip Page
->Purpose: Initiate the creation of a new travel plan.
->Functionality:
->Trip name input
->Travel start and end date selection
->Basic trip description
->Submission flow to begin trip planning
